<p class="has-line-data" data-line-start="15" data-line-end="16">La arquitectura cliente-servidor es un modelo informático donde un cliente solicita servicios y un servidor los proporciona. Es la base de cómo funcionan la mayoría de aplicaciones y páginas web que usamos a diario.</p>
<p class="has-line-data" data-line-start="17" data-line-end="18">Concepto básico:</p>
<ul>
<li class="has-line-data" data-line-start="19" data-line-end="21">
<p class="has-line-data" data-line-start="19" data-line-end="20">Cliente: Es el dispositivo o aplicación que inicia la petición. Por ejemplo, tu navegador web cuando entras en una página.</p>
</li>
<li class="has-line-data" data-line-start="21" data-line-end="23">
<p class="has-line-data" data-line-start="21" data-line-end="22">Servidor: Es el sistema que recibe la petición, procesa la información y devuelve la respuesta. Por ejemplo, el servidor de Google que te entrega los resultados de búsqueda.</p>
</li>
<li class="has-line-data" data-line-start="23" data-line-end="25">
<p class="has-line-data" data-line-start="23" data-line-end="24">Comunicación: Se realiza mediante protocolos de red (como HTTP), que permiten que ambos se entiendan y trabajen juntos.</p>
</li>
</ul>
<p class="has-line-data" data-line-start="25" data-line-end="26">Funcionamiento:</p>
<ol>
<li class="has-line-data" data-line-start="27" data-line-end="28">El cliente envía una solicitud (ejemplo: abrir una web).</li>
<li class="has-line-data" data-line-start="28" data-line-end="29">El servidor recibe la solicitud, la procesa y busca los datos necesarios.</li>
<li class="has-line-data" data-line-start="29" data-line-end="31">El servidor responde al cliente con la información solicitada (ejemplo: el contenido de la página).</li>
</ol>
<p class="has-line-data" data-line-start="31" data-line-end="32">Ejemplos cotidianos:</p>
<ul>
<li class="has-line-data" data-line-start="33" data-line-end="34">Navegar por Internet: tu navegador (cliente) pide páginas a un servidor web.</li>
<li class="has-line-data" data-line-start="34" data-line-end="35">Apps móviles: la app pide datos a un servidor en la nube.</li>
<li class="has-line-data" data-line-start="35" data-line-end="37">Correos electrónicos: el cliente de correo solicita mensajes al servidor de correo.</li>
</ul>
<p class="has-line-data" data-line-start="37" data-line-end="38">Ventajas:</p>
<ul>
<li class="has-line-data" data-line-start="39" data-line-end="40">Centralización: el servidor concentra los datos y procesos.</li>
<li class="has-line-data" data-line-start="40" data-line-end="41">Escalabilidad: se pueden añadir más clientes sin cambiar el servidor.</li>
<li class="has-line-data" data-line-start="41" data-line-end="43">Seguridad: el servidor controla accesos y permisos.</li>
</ul>
<p class="has-line-data" data-line-start="43" data-line-end="44">Inconvenientes:</p>
<ul>
<li class="has-line-data" data-line-start="45" data-line-end="46">Dependencia del servidor: si falla, los clientes no pueden acceder.</li>
<li class="has-line-data" data-line-start="46" data-line-end="48">Carga: muchos clientes simultáneos pueden saturar el servidor.</li>
</ul>
<p class="has-line-data" data-line-start="48" data-line-end="49">En resumen, el cliente-servidor es como una conversación organizada entre dos partes: uno pide y otro responde. Es un modelo simple pero esencial para entender cómo funciona Internet y las aplicaciones modernas.</p>
